# 交互系统化迭代设计

## 背景

本轮迭代来自连续测试后的体验反馈。现有页面已经完成多空间、今日重点、保存状态、移动端分区等能力，但菜单、确认、消息、快捷键、图片预览和文本编辑行为仍分散在不同组件中。继续以单点补丁方式迭代，会让用户感知和代码维护都变得不稳定。

本设计选择长期方案：把菜单、确认、消息、快捷键、拖放和图片预览沉淀为统一交互底座，再在待办、图片、快捷链接、设置、文本编辑和工作空间 tab 中应用这些规则。

## 目标

1. 减少界面常驻按钮和视觉噪音。
2. 建立统一的对象操作模型：区域操作进三个点，对象操作进右键菜单。
3. 删除类动作统一走确认，不再提供删除后撤销。
4. 所有消息气泡文案和颜文字集中管理，方便后续维护。
5. 提升图片区的桌面化效率：拖入、预览、快捷键和右键菜单一致可用。
6. 让文本编辑更接近自然笔记体验，支持撤销和缩进取消。
7. 修复本轮测试暴露的图片预览、气泡定位、导入确认和 tab 滚动问题。

## 统一交互底座

### 消息系统

- 所有气泡文案、颜文字、类型、时长和用途集中到一个 TypeScript 文案文件管理。
- 文案文件需要写清楚注释，方便后续开发者手动更新消息内容。
- 组件只传消息场景或消息 key，不直接硬编码气泡正文。
- 删除后不再显示撤销按钮，也不保留删除后撤销能力。
- 删除完成后只显示结果反馈。
- 关于、导入确认、删除确认、清理确认、成功提示、失败提示都走统一消息或确认模型。
- 关于气泡文字不做硬性长度限制，但仍应保持可读。
- GIF 动态头像每次出现最多 10 秒，超过 10 秒自动淡出；新消息出现时可以重置计时。

### 菜单系统

- 区域级操作进入区域标题旁的三个点菜单。
- 对象级操作进入该对象的右键菜单。
- 菜单文案保持短句，避免长说明。
- 同一对象在普通态和预览态复用同一套菜单定义，只根据状态过滤不适合的菜单项。
- 待办、快捷链接、图片、预览图片、工作空间 tab 的菜单都遵守同一套命名和分工规则。

### 确认系统

- 删除图片、删除快捷链接、删除提醒事项、清理已完成提醒事项、删除工作空间都需要二次确认。
- 每种确认都必须明确说明动作后果。
- 确认按钮使用语义化文案，例如“删除”“清理”“删除空间”。
- 普通文本编辑不弹删除确认，交给编辑器自身撤销能力处理。
- 删除后不可撤销，确认气泡不能暗示仍可恢复。

### 快捷键系统

- 图片预览态：
  - `Enter` 复制当前图片。
  - `Delete` 或 `Backspace` 删除当前图片，删除前确认。
  - `Escape` 关闭预览。
- 文本编辑态：
  - `Ctrl+Z` 撤销当前文本区编辑。
  - `Ctrl+S` 立即保存。
  - 空行在行首按删除键或退格键时优先取消缩进。
- 快捷键只在对应区域或状态生效，避免全局互相抢事件。

### 拖放系统

- 图片区统一处理浏览器外部图片拖入。
- 支持一次拖入多张图片。
- 非图片文件自动忽略。
- 拖入 1 张图片时，添加到图片列表并自动复制这张图片。
- 拖入多张图片时，全部添加到图片列表并自动复制最后一张。
- 部分文件不是图片时，添加有效图片并提示已忽略非图片。
- 全部文件都不是图片时，不添加并提示没有可添加的图片。
- 读取失败、存储失败、复制失败和成功反馈都走统一消息系统。

## 待办区域

### 默认展示

- 初始默认隐藏已完成提醒事项。
- 显示或隐藏已完成的状态需要持久保存，下次打开沿用上次选择。
- 每个时间段标题旁继续显示进度，格式为“已完成 / 总数”。
- 隐藏已完成时，进度仍统计全部事项，而不是只统计可见事项。
- 如果某个时间段只有已完成事项且当前隐藏已完成，该时间段列表区域显示轻量空态，不显示已完成项。

### 今日重点

- 今日重点显示星标提醒事项。
- 未完成星标正常显示，已完成星标弱化显示。
- 今日重点中的星标放在提醒事项最右侧，和普通时间段保持一致。
- 在今日重点中勾选完成时增加短动画，例如完成态过渡、轻微淡出或勾选反馈。
- 即使当前隐藏已完成，今日重点仍可以弱化显示已完成星标事项，方便用户回顾重点完成情况。

### 单击编辑和新增

- 单击已有提醒事项文本进入编辑。
- 单击时间段内的空白列表区域新增提醒事项。
- 单击标题、勾选框、星标、拖动区域、三个点菜单或右键菜单时，不触发新增或编辑。
- 如果当前已有正在编辑的提醒事项，先保存当前编辑，再进入新的编辑或新增。

### 区域三个点菜单

待办区域三个点菜单只放区域级功能，不再提供“新增提醒事项”。

菜单项：

- `显示已完成` / `隐藏已完成`
- `清理已完成`

清理已完成需要确认。确认后删除已完成事项，不提供撤销。

### 提醒事项右键菜单

菜单文案保持简短：

- `复制`
- `编辑`
- `删除`
- `星标` / `取消星标`
- 可选移动项：`移到早上`、`移到中午`、`移到晚上` 等短文案

删除单条提醒事项需要确认。复制时复制该提醒事项文本。

### 反馈位置

- 如果待办区域未聚焦，勾选提醒事项后的消息气泡应出现在待办区域右下角。
- 删除、清理、复制等反馈尽量锚定到待办区域。

## 图片区与预览

### 外部拖入图片

- 支持从浏览器外部拖入一张或多张图片到图片区。
- 只接收图片文件，非图片文件自动忽略。
- 拖入 1 张时，添加并自动复制该图片。
- 拖入多张时，全部添加并自动复制最后一张。
- 全部不是图片时，提示没有可添加的图片。
- 图片读取、存储和复制过程中的失败都要给出消息反馈。

### 预览快捷键

- 首次打开图片预览后，`Enter` 必须立即复制当前图片。
- `Delete` 或 `Backspace` 删除当前预览图片，删除前确认。
- `Escape` 关闭预览。
- 预览打开后，当前图片必须立即成为快捷键目标，不需要先切换图片。

### 预览点击行为

- 点击图片外空白区域关闭预览。
- 点击图片本身不关闭。
- 点击菜单、确认气泡、右键菜单时不误关闭预览。

### 右键菜单

缩略图右键菜单：

- `复制`
- `预览`
- `删除`

预览态右键菜单：

- `复制`
- `取消预览`
- `删除`

预览态不显示“预览”，普通态不显示“取消预览”。

### 删除反馈

- 删除图片需要确认。
- 删除后不提供撤销。
- 删除成功或失败气泡出现在图片区右下角，不出现在整页右下角。
- 如果正在预览的图片被删除，预览自动关闭。
- 如果删除失败，保留预览并提示失败。

## 快捷链接

### 区域三个点菜单

快捷链接区域不再常驻多个功能按钮，区域标题旁提供三个点菜单。

菜单项：

- `新增`
- `显示隐藏项` / `隐藏隐藏项`

显示或隐藏状态需要持久保存，下次打开沿用上次选择。

### 复制文本

- 文本型快捷链接复制时保留原始换行。
- 链接型快捷链接保持既有打开或复制逻辑。
- 复制成功和失败都走统一消息气泡。

### 右键菜单

菜单文案保持简短：

- `复制`
- `编辑`
- `隐藏` / `显示`
- `删除`

删除快捷链接需要确认。删除后不提供撤销，只显示结果气泡。

## 设置与关于

- 设置菜单中的“关于”不再打开弹框，改为显示消息气泡。
- 关于气泡可以包含项目名、版本和简短说明。
- 关于气泡文字长度不做硬性限制，但不应写成长段说明。
- “提建议”继续跳转到 GitHub issue 创建页面。
- 设置菜单和保存状态不应与工作空间 tab 重叠。

## 文本编辑器与工作空间 tab

### 工作空间 tab 滚动

- 当 tab 总宽度没有超过容器时，不展示横向滚动条。
- 当 tab 超过最大宽度时，展示可用的横向滚动。
- Windows 上不应出现空滚动条占位区域。
- 鼠标滚轮、触控板横向滚动和拖动滚动条都应可用。
- 设置和保存状态区域不能与 tab 重叠。

### 文本编辑撤销

- 便签、工作空间、存储空间的文本编辑支持 `Ctrl+Z` 撤销。
- 撤销范围限定在当前文本区，不跨区域互相影响。
- 用户输入、换行、缩进、取消缩进、删除行都应进入撤销栈。
- 普通文本编辑删除不需要确认。

### 删除键取消缩进

- 如果某一行文字已被删空，再按 `Delete` 或 `Backspace` 可以取消该行缩进。
- 该规则覆盖便签、工作空间、存储空间。
- 行没有缩进时，再按删除键或退格键继续执行原有合并或删除行为。

### 工作空间 tab 右键菜单

菜单项：

- `编辑`
- `删除`

删除空间需要确认。只剩最后一个空间时，`删除` 禁用或不显示。空间删除后不可恢复，不提供撤销。

## 必须修复的 bug

1. 首次预览图片后按 `Enter` 不能复制图片。
2. 删除图片后气泡出现在页面右下角，而不是图片区右下角。
3. 数据导入选择文件但取消确认后，第二次选择文件无法弹出确认气泡。
4. 工作空间 tab 不溢出时在 Windows 上出现空横向滚动条区域。
5. 工作空间 tab 过多时横向滚动不可用或难以使用。

## 验收标准

### 待办

- 默认隐藏已完成提醒事项。
- 区域三个点菜单只有“显示/隐藏已完成”和“清理已完成”。
- 单击事项文本进入编辑。
- 单击列表空白区域新增事项。
- 事项右键菜单使用短文案，例如“复制”“编辑”“删除”“星标”。
- 清理已完成有确认气泡，删除后无撤销。
- 今日重点勾选有动画，星标在右侧。

### 图片

- 外部拖入一张图片会添加并自动复制。
- 外部拖入多张图片会全部添加并自动复制最后一张。
- 非图片文件被忽略并提示。
- 预览态支持 `Enter` 复制、`Delete` 或 `Backspace` 删除、`Escape` 关闭。
- 预览态右键菜单包含“复制”“取消预览”“删除”。
- 点击空白关闭预览。
- 删除图片前确认，删除后无撤销。

### 快捷链接

- 区域功能收进三个点菜单。
- 菜单包含“新增”和“显示/隐藏隐藏项”。
- 文本复制保留换行。
- 右键菜单文案简短。
- 删除前确认，删除后无撤销。

### 文本编辑

- 文本区支持 `Ctrl+Z` 撤销。
- 空行再次按删除键或退格键可取消缩进。
- 规则覆盖便签、工作空间、存储空间。
- `Ctrl+S` 和失焦保存继续可用。

### 消息系统

- 所有气泡文案和颜文字集中到一个 TypeScript 文件。
- 删除撤销相关文案、按钮和能力被移除。
- 关于从弹框改为气泡。
- GIF 每次出现最多 10 秒，超过后自动淡出。
- 重要反馈位置贴近触发区域。

### 回归

- 导入和导出仍可用。
- 保存状态仍正确显示。
- 移动端菜单不和右上角按钮重叠。
- 工作空间 tab 多时可滚动，少时不显示空滚动条。
- 现有测试需要补齐或更新，覆盖核心交互。
