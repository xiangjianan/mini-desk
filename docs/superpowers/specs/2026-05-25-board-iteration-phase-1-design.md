# 看板第一包体验迭代设计

## 背景

本轮需求拆成小版本推进。第一包优先处理低结构风险、高体验收益的事项，暂不重构提醒事项列表模型，也不实现系统级通知和 GIF 主题。这样可以尽快修复现有工作流中的编辑、拖拽、时间显示和误触问题，并为后续通知能力和可配置列表保留清晰边界。

## 范围

第一包包含：

1. 外部文本拖拽到工作空间和提醒事项。
2. 文本编辑器统一支持数字编号自动渲染和重排。
3. 截止时间语义改为通知时间，且与星标解耦。
4. 通知时间弹窗改为左侧日期、右侧圆形快捷时间布局。
5. 提醒事项长文本不被空通知时间区域覆盖。
6. 页面重新聚焦时刷新通知时间状态。
7. 工作空间等文本区在选中文本后立即进入编辑模式。
8. 快捷按钮编辑弹窗的删除按钮改为取消按钮。
9. 新建工作空间后直接编辑空间名。
10. 已完成分隔行增加 clear 按钮。

第一包不包含：

- Web Notification API 系统级通知。
- GIF 主题选择和关闭 GIF 的设置。
- 提醒事项列表新增、删除、折叠、收缩和拖拽排序。

## 总体方案

采用小步补齐现有结构的方案。当前早上、中午、晚上三组提醒事项保持不变，第一包只改现有组件行为、状态字段语义和局部样式。可配置提醒事项列表留到第三包，避免在本轮同时改动排序、今日重点、已完成显示和列表持久化结构。

## 通知时间

### 数据模型

给 `TodoItem` 增加可选字段：

```ts
notifyAt?: number
```

`notifyAt` 表示通知时间的本地毫秒时间戳。旧数据中的 `deadlineAt` 作为兼容输入：加载或导入时，如果 `notifyAt` 不存在且 `deadlineAt` 是有效数字，则迁移为 `notifyAt`。保存时只写出 `notifyAt`，不再写出 `deadlineAt`。

星标与通知时间解耦。星标继续表示“今日重点 / 关注”，通知时间可以设置在任意提醒事项上。取消星标不会清除通知时间；取消通知时间只清除 `notifyAt`。

### 交互

所有提醒事项都可以打开通知时间弹窗。入口沿用任务右侧操作区，但文案从“截止时间”改为“通知时间”。如果任务已有通知时间，入口文案为“编辑通知时间”；否则为“设置通知时间”。

弹窗采用双栏布局：

- 左侧直接选择日期。
- 右侧显示圆形快捷时间，按钟表方向排列。
- 快捷时间 hover 时放大，但不改变整体弹窗尺寸。
- 底部按钮为“不设通知时间”和“确定”。
- 关闭弹窗不修改任务。

“不设通知时间”对任意任务都有效：如果任务已有 `notifyAt`，点击后清除；如果没有，则保持无通知时间。

### 显示与刷新

提醒事项和今日重点中都显示通知时间标签。没有 `notifyAt` 时不渲染通知时间占位区域，避免长文本被空白区域覆盖。

通知时间状态包括已超期、24 小时内、3 天内和更晚，沿用现有颜色层级，但文案改为通知语义。已完成任务不显示紧急底色。

组件继续每分钟刷新一次当前时间。页面从后台回到前台时，通过 `window.focus` 和 `visibilitychange` 立即更新当前时间，确保超期、背景色和标签立刻重算。

### 排序

第一包不因通知时间改变任务排序。当前星标、完成态和用户手动顺序的规则保持稳定。通知时间只影响标签、提醒状态和后续系统通知能力。

## 文本编辑器

备忘录、工作空间和工程文件共用同一套文本编辑器编号规则。

数字编号自动渲染采用“数据不写死编号”的方案：状态中仍保存 `LineItem.text` 和 `indent`，编辑器显示时识别有序列表段并渲染连续编号。删除中间行后，后续行按当前顺序重新编号。

第一包只识别低误判场景：

- 同一缩进层级。
- 从 `1.` 开始。
- 连续有序列表行。

普通版本号、日期和散落数字不会被当作有序列表。按 Enter 在编号行中继续下一行编号；在空编号行按 Enter 或删除时退出编号段。

文本区在非编辑状态下选中文本时立即进入编辑模式并解除 readonly。选中后可以直接剪切、粘贴和继续输入。该行为适用于所有 `TextPanel` 使用处，包括备忘录、工作空间和工程文件。

## 外部文本拖拽

外部文本拖拽读取 `dataTransfer.getData("text/plain")`。内部提醒事项拖拽优先级更高；如果存在内部拖拽状态，则继续执行现有移动逻辑，不当作外部文本。

拖到工作空间文本区域时，将文本追加到当前工作空间末尾。已有内容非空时先补一个换行，拖入文本保留多行结构并走统一的 `LineItem` 序列化逻辑。

拖到提醒事项列表区域时，新增提醒事项。多行文本按行拆成多个提醒事项，空行忽略。单行文本新增一个提醒事项。新增任务插入当前列表未完成任务区域，保持现有添加逻辑。

## 工作空间

新建工作空间后立即进入新空间标签名编辑状态。默认名全选，用户可以直接输入。回车或失焦提交；Esc 取消编辑并保留默认名称。

该行为只改变创建后的焦点，不改变现有双击标签编辑、右键编辑、删除和拖拽排序能力。

## 快捷按钮

快捷按钮编辑弹窗中的“删除”按钮改为“取消”。点击取消只关闭弹窗，不修改数据。删除入口继续保留在右键菜单中，并沿用现有二次确认，降低弹窗内误删风险。

## 已完成清理

提醒事项列表出现“已完成”分隔行时，在分隔行右侧显示 `clear` 按钮。点击后只清理当前列表已完成任务，复用现有 `clearCompleted` 状态逻辑和二次确认文案。

没有已完成任务时不显示分隔行，也不显示 clear 按钮。已完成列表被隐藏时不显示分隔行和 clear 按钮；用户仍可通过列表菜单执行现有清理动作。

## 组件边界

### `src/types.ts`

- 给 `TodoItem` 增加 `notifyAt?: number`。
- 保留 `deadlineAt` 只作为兼容输入类型时的内部读取，不作为新状态输出字段。

### `src/state/storage.ts`

- 导入和加载时归一化 `notifyAt`。
- 兼容旧 `deadlineAt`。
- 序列化时写出 `notifyAt`。

### `src/state/deadlines.ts`

- 重命名或新增通知时间语义函数。
- 保留兼容 wrapper 时，内部使用通知时间命名。
- 生成通知时间标签、默认日期时间和快捷时间。

### `src/state/todos.ts`

- 增加设置和清除 `notifyAt` 的纯函数。
- 星标函数不再清理通知时间。
- 排序不因 `notifyAt` 改变。

### `src/components/TodoPanel.vue`

- 打开、确认、清除通知时间弹窗。
- 渲染双栏日期 / 圆形时间布局。
- 长文本和通知标签使用自适应布局。
- 页面聚焦刷新当前时间。
- 外部文本 drop 新增提醒事项。
- 已完成分隔行显示 clear 按钮。

### `src/components/TextPanel.vue`

- 支持外部文本 drop 追加。
- 选中文本时进入编辑模式。
- 使用统一编号渲染和保存逻辑。

### `src/components/SpacePanel.vue`

- 新建空间后进入标签编辑状态。
- 将外部文本追加事件转发到当前空间。

### `src/components/QuickButtons.vue`

- 编辑弹窗删除按钮改为取消按钮。

## 错误和边界

- 拖入空文本不创建内容。
- 外部多行文本中的空行不创建空提醒事项。
- 内部提醒事项拖拽不触发外部文本新增。
- 通知时间弹窗关闭不保存半成品。
- 选择过去时间是允许的，保存后立即显示超期。
- 清除通知时间不影响星标和完成状态。
- 取消星标不影响通知时间。
- 旧数据中的 `deadlineAt` 非法时丢弃。
- 编号识别只处理从 `1.` 开始的连续段，避免误处理日期和版本号。

## 测试计划

状态测试：

- 旧 `deadlineAt` 加载后迁移为 `notifyAt`。
- 新状态保存时包含 `notifyAt`，不依赖 `deadlineAt`。
- 任意提醒事项可以设置和清除 `notifyAt`。
- 取消星标不清除 `notifyAt`。
- 设置 `notifyAt` 不改变当前排序。
- 多行外部文本可拆成多个提醒事项。
- 外部文本可追加到当前工作空间末尾。

文本编辑测试：

- 从 `1.` 开始的连续编号删除中间行后自动重排。
- 不同缩进层级编号互不影响。
- 日期和版本号不被识别为编号列表。
- 编号行按 Enter 继续下一编号。
- 空编号行退出编号段。

组件测试：

- 通知时间弹窗显示日期区、圆形快捷时间区、“不设通知时间”和“确定”。
- hover 快捷时间产生放大样式且不撑开布局。
- 没有通知时间时长文本不被右侧空白覆盖。
- 页面 focus 或 visibilitychange 后通知标签刷新。
- 文本区选中文本后可以直接剪切和粘贴。
- 新建工作空间后空间名输入框自动聚焦并全选。
- 快捷按钮编辑弹窗中的取消按钮只关闭弹窗。
- 已完成分隔行 clear 按钮清理当前列表已完成任务。
