# 移动端桌面引导设计

## 背景

当前 To Do List 看板是桌面端优先的本地工作台。桌面端布局依赖多列并排、拖拽排序、图片预览、快捷键、右键菜单、文本编辑和工作空间 tab 等高密度交互。

项目之前为移动端提供了区域抽屉切换和触控编辑适配，但移动端体验仍然不符合这个产品的核心使用场景。继续维护移动端各区域功能会增加交互复杂度，也容易让用户以为移动端应具备完整能力。

本次设计将移动端策略调整为：移动端访问时完全阻断各业务区域功能，但用友好的提示页和消息气泡承接用户，建议其在桌面端浏览器打开完整工作台。目标不是劝退用户，而是让移动端打开页面时看到清晰、轻量、体面的状态。

## 目标

1. 移动端不再提供图片、便签、快捷链接、待办、工作空间等区域功能。
2. 移动端不再提供区域抽屉切换。
3. 移动端显示友好的桌面端引导，不表现为错误页或不可用页面。
4. 桌面端现有布局、状态、交互和数据保存逻辑不受影响。
5. 保留最小必要的全局外观能力，例如主题切换。
6. 更新测试，明确移动端是阻断式提示页，而不是功能分区页面。

## 非目标

- 不开发移动端完整工作台。
- 不提供移动端只读数据概览。
- 不提供移动端编辑、拖拽、导入导出、图片预览或区域切换能力。
- 不改变桌面端的看板信息架构。
- 不迁移或重写现有业务状态模型。

## 体验设计

### 桌面端

桌面端保持现状：页面继续渲染完整看板，包括图片区、便签、快捷链接、待办区域、工作空间、设置入口、保存状态、主题切换和消息气泡。

桌面端不显示移动端引导页。

### 移动端

移动端断点沿用当前 `max-width: 900px`。在该断点内，页面进入移动端阻断状态。

移动端只显示一个轻量引导界面：

- 项目名称：`To Do List 看板`
- 简短说明：这是为桌面端工作流设计的本地优先工作台。
- 主要提示：`建议在电脑浏览器打开，以获得完整体验 (｡•̀ᴗ-)✧`
- 视觉上保留线框、紧凑、安静的项目风格。

移动端可以继续显示主题切换按钮，便于用户在浅色和暗色环境下阅读提示页。设置、导入、导出、关于、提建议等入口在移动端隐藏，避免暴露业务操作。

### 消息气泡

移动端引导应复用现有 `CompanionBubble` 的视觉语言。气泡承载桌面端建议文案，使移动端页面看起来像被助手温和提醒，而不是被系统强制拦截。

气泡不需要确认按钮，不需要外链，不需要自动打开任何新页面。用户进入移动端页面后能自然看到提示即可。

## 架构与组件

### App 壳层

`App.vue` 负责根据视口状态决定展示完整看板还是移动端引导壳。视口状态应通过 `window.matchMedia("(max-width: 900px)")` 或等价的响应式封装获得，并在窗口尺寸变化时更新。

建议增加一个明确的状态命名，例如 `isMobileBlocked` 或同类计算状态，用于表达当前处于移动端阻断体验。该状态只决定渲染壳层，不改变 `BoardState` 数据。

桌面端继续渲染现有 `<main class="board">` 内的业务区域。

移动端渲染新的引导容器，例如：

- `main.mobile-handoff`
- 标题区域
- 简短说明
- companion 气泡
- 主题切换按钮

旧的 `mobileAreas`、`mobileActiveArea`、`mobileNavOpen` 和移动端抽屉菜单不再需要。实现时应删除这组状态和模板，避免保留不可达功能。

### CompanionBubble

优先复用现有组件，不新增第二套气泡组件。移动端可以传入固定的提示消息和固定位置。

如果现有气泡生命周期不适合常驻移动端提示，可以在 `App.vue` 层用单独的移动端文案和可见状态驱动它，避免改动组件内部动画逻辑。只有当现有组件无法稳定展示移动端提示时，才对 `CompanionBubble` 增加小范围 prop。

### 样式

`styles.css` 中移动端媒体查询需要从“分区工作台布局”改为“移动端引导页布局”。

应删除或替换以下移动端职责：

- `.mobile-nav`
- `.mobile-drawer-trigger`
- `.mobile-drawer-menu`
- `.mobile-menu-option`
- `.mobile-banner`
- `data-mobile-active` 驱动的区域显示规则

新增移动端引导页样式时，应保持：

- `100dvh` 视口高度。
- 不允许页面横向滚动。
- 文案不与顶部操作或气泡重叠。
- 暗色主题变量可自然生效。
- 不使用营销式大 hero，不引入装饰性渐变或复杂插图。

## 数据流

移动端阻断状态不修改用户数据。

页面仍可以正常加载 `localStorage` 和 IndexedDB 中的已有数据，但移动端不展示业务区域，也不触发编辑、拖拽、导入导出、图片复制、图片预览等业务操作。

主题切换仍写入现有主题状态，并沿用既有持久化逻辑。

## 键盘与事件

移动端提示页不需要支持业务快捷键。实现时应避免移动端触发以下行为：

- `Ctrl+S` 显示保存气泡。
- 图片预览快捷键。
- 粘贴图片到图片区。
- 文本编辑聚焦提示。

桌面端仍保留这些行为。

## 错误处理

移动端引导页本身没有复杂错误路径。若气泡资源加载失败，页面仍应显示文字提示。

主题切换失败或持久化失败不新增错误提示，沿用当前应用行为。

## 测试计划

### 单元与渲染测试

更新现有移动端测试：

- 不再断言移动端抽屉菜单存在。
- 断言移动端引导容器存在。
- 断言移动端文案包含桌面端建议。
- 断言移动端不渲染图片区、便签、快捷链接、待办和工作空间功能入口。
- 断言移动端隐藏设置入口，保留主题切换入口。
- 断言桌面端仍渲染完整看板区域。

### 静态契约测试

更新 `naive-components` 中关于移动端分区布局的静态断言：

- 移除对 `data-mobile-active`、`.mobile-nav`、`.mobile-drawer-menu` 的要求。
- 增加对移动端引导页样式和阻断式区域隐藏规则的要求。

### 手动验证

在桌面宽度打开页面：

- 完整看板可见。
- 原有区域功能可用。
- 主题切换可用。

在移动端宽度打开页面：

- 只看到移动端引导页。
- 不出现区域抽屉。
- 不出现业务区域内容。
- 气泡或提示文案清楚建议在桌面端打开。
- 暗色主题下文字仍清晰。
