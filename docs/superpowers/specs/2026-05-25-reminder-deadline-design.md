# 星标截止时间提醒设计

## 背景

现有提醒事项已经按早上、中午、晚上三个时间段组织，并支持星标重点、今日重点、完成隐藏、拖拽排序和二次确认。新的提醒能力需要在不打散当前工作流的前提下，让临近交付时间的任务更靠前、更明显。

本设计把截止时间绑定到星标任务上：星标代表任务重要，截止时间代表重要任务的交付压力。普通任务保持轻量，不强制增加时间字段。

## 目标

1. 点亮星标时可以顺手设置截止日期和常用整点。
2. 只在星标任务上显示和使用截止时间。
3. 每个时间段内部按截止时间临近程度排序，不打散早上、中午、晚上三个框。
4. 越临近截止时间，提醒越明显；超期需要更强提醒。
5. 取消星标需要二次确认，确认后同步清除截止时间。
6. 旧数据保持兼容，没有截止时间的历史任务继续正常使用。

## 非目标

- 不做进度条、百分比或工作量拆分。
- 不做分钟、秒级时间选择。
- 不做日历视图、全局时间线或跨时间段自动重排。
- 不做浏览器通知、系统通知或后台定时提醒。
- 不从任务文字中自动解析日期。

## 用户交互

### 点亮星标

1. 用户点击未星标任务右侧的 `☆`。
2. 系统立即打开日期和时间选择面板，不再先询问是否设置截止时间。
3. 面板提供日期选择和常用整点按钮。
4. 用户选择日期和时间后点击 `确定`，任务同时变为星标并保存截止时间。
5. 用户点击 `忽略`，任务只变为星标，不保存截止时间。
6. 用户直接关闭面板，任务不变，避免误点留下半成品。

### 时间选择

时间精度只到小时。默认提供常用整点按钮：

- `09:00`
- `12:00`
- `15:00`
- `18:00`
- `21:00`

日期必须选择。时间如果用户不主动选择，默认使用 `18:00`，因为它更接近日常交付结束时间，也避免确认按钮因为缺少时间而不可用。

### 取消星标

1. 用户点击已星标任务右侧的 `★`。
2. 系统弹出确认气泡。
3. 如果任务有截止时间，确认文案需要说明取消重点会同步清除截止时间。
4. 用户点击 `取消重点` 后，系统取消星标并删除 `deadlineAt`。
5. 用户点击 `算了` 或关闭气泡，任务保持不变。

确认按钮固定：

- 主按钮：`取消重点`
- 次按钮：`算了`

有截止时间时，确认文案从以下 10 条中随机选择：

1. `取消重点后，截止时间也会一起清掉。`
2. `这个任务将不再置顶提醒，截止时间也会移除。`
3. `确定取消重点吗？已设置的截止时间会同步清除。`
4. `取消后，它会回到普通提醒事项，截止时间不再保留。`
5. `这个重点要撤掉吗？截止时间也会一起撤掉。`
6. `取消重点会停用这条任务的截止提醒。`
7. `确认取消重点？相关截止时间也会删除。`
8. `取消后不再按截止时间提醒这个任务。`
9. `这条任务会变回普通事项，截止时间会清掉。`
10. `确定不再重点关注它吗？截止时间也会一并移除。`

没有截止时间时，使用同一批文案的短版本，不提截止时间：

1. `确定取消重点吗？`
2. `这个任务将不再置顶提醒。`
3. `确定不再重点关注它吗？`
4. `取消后，它会回到普通提醒事项。`
5. `这个重点要撤掉吗？`
6. `取消重点后，它会按普通任务显示。`
7. `确认取消这个重点？`
8. `取消后不再把它放进今日重点。`
9. `这条任务会变回普通事项。`
10. `确定撤下这个重点任务吗？`

## 排序规则

排序只在每个时间段内部生效。早上、中午、晚上三个时间段不会互相打散。

每个时间段内部的顺序为：

1. 未完成、已星标、且有截止时间的任务，按 `deadlineAt` 从近到远排序。
2. 未完成、已星标、但没有截止时间的任务，排在有截止时间的星标任务后面。
3. 未完成、未星标的普通任务，保持用户手动顺序。
4. 已完成任务仍放在底部，沿用现有已完成分组和隐藏逻辑。

如果多个任务的截止时间相同，按原列表顺序保持稳定排序。拖拽仍用于调整同一优先级组内的相对顺序；有截止时间的星标任务会在下次渲染时继续按截止时间排序。

## 提醒样式

采用“截止时间标签 + 左侧警示线”的方案，保持当前紧凑列表形态。

### 展示位置

在任务文本和星标按钮之间增加截止时间标签。示例：

- `! 已超期`
- `! 今天 18`
- `! 明天 18`
- `3天后 18`
- `5/30 18`

今日重点中的任务也显示同样的截止时间标签。

### 提醒等级

以当前时间和 `deadlineAt` 的差值计算提醒等级：

1. 已超期：红色左线、浅红底、标签为 `! 已超期`。
2. 24 小时内：红色左线，标签带 `!`，例如 `! 今天 18` 或 `! 明天 18`。
3. 3 天内：黄色左线，标签例如 `3天后 18`。
4. 3 天以上：灰色左线，标签例如 `5/30 18`。

已完成任务不再显示紧急底色，避免已完成内容继续制造压力；如显示已完成列表，截止时间标签可以保留为弱化样式。

## 数据模型

给 `TodoItem` 增加一个可选字段：

```ts
deadlineAt?: number
```

含义是截止时间的毫秒时间戳。没有该字段表示没有截止时间。

示例：

```ts
{
  id: "abc",
  text: "交付视觉稿",
  done: false,
  starred: true,
  deadlineAt: 1779722400000
}
```

状态持久化仍写入现有 `todo-board-state-v1`。导入和加载时只接受有效数字时间戳；非法值、空值或非数字值会被丢弃。

取消星标时必须同步删除 `deadlineAt`，让数据语义保持一致。

## 组件边界

### `TodoPanel.vue`

负责：

- 打开星标截止时间面板。
- 展示日期和常用整点选择。
- 展示截止时间标签和提醒等级样式。
- 发出星标、设置截止时间、取消星标请求。
- 在取消星标前触发确认流程。

### `src/state/todos.ts`

负责：

- 支持同时设置 `starred` 和 `deadlineAt`。
- 取消星标时清理 `deadlineAt`。
- 按新的优先级规则排序。
- 保持排序稳定性。

### `src/state/storage.ts`

负责：

- 加载和导入时归一化 `deadlineAt`。
- 保持旧数据兼容。

### `src/state/messages.ts`

负责：

- 管理取消星标确认文案。
- 区分有截止时间和无截止时间两种文案池。

## 错误和边界处理

- 直接关闭日期时间面板时不修改任务。
- 日期未选择时，`确定` 不生效或保持禁用。
- 时间未选择时默认使用 `18:00`。
- 选择过去时间是允许的，保存后立即进入超期提醒状态。
- 取消星标确认期间，任务数据不提前变化。
- 已完成任务即使有历史 `deadlineAt`，也不参与未完成排序。
- 旧任务没有 `deadlineAt` 时按普通星标或普通任务处理。

## 测试计划

状态测试：

- 点亮星标并确认时间后，同时写入 `starred: true` 和 `deadlineAt`。
- 点击忽略后，只写入 `starred: true`。
- 取消星标后，`starred` 变为 `false` 且 `deadlineAt` 被移除。
- 有截止时间的星标任务按截止时间从近到远排序。
- 无截止时间星标任务排在有截止时间星标任务之后、普通任务之前。
- 已完成任务仍在底部。
- 导入旧数据时没有 `deadlineAt` 也能正常加载。
- 导入非法 `deadlineAt` 时丢弃该字段。

组件测试：

- 点击 `☆` 打开日期时间选择面板。
- 点 `确定` 发出星标和截止时间更新。
- 点 `忽略` 只发出星标更新。
- 关闭面板不发出更新。
- 点击 `★` 打开取消星标确认气泡。
- 有截止时间时，确认文案来自截止时间文案池。
- 无截止时间时，确认文案来自普通取消文案池。
- 确认取消后清除星标和截止时间。
- 超期、24 小时内、3 天内、3 天以上任务显示对应 class 和标签。
