# To Do List 看板 UX 优化需求

## 背景

当前看板采用五栏全屏布局：图片、便签与快捷按钮、提醒事项、工作空间、工程文件。这个布局适合桌面端长期打开，但在持续使用中暴露出几个体验问题：

- 提醒事项区域的信息层级偏平，重点任务、完成进度和已完成内容不够清晰。
- 部分高频操作依赖右键或显性较弱的控件，新用户发现成本较高。
- 工作空间和工程文件都以大文本区域呈现，右侧 50% 屏幕职责重复。
- 保存反馈有趣但缺少稳定、明确的全局保存状态。
- 移动端不适合强行承载桌面五栏布局。

本需求只描述用户体验目标和行为，不约束具体技术实现。

## 目标

1. 让用户打开页面后更快知道今天最重要的事项和当前进展。
2. 减少提醒事项行内控件的视觉占用，让列表更紧凑、更清爽。
3. 建立统一的编辑心智模型，降低学习成本。
4. 将右侧两个相似文本区整合为可扩展的多空间工作区。
5. 为桌面和移动端分别提供符合使用场景的布局体验。
6. 让保存状态明确可见，避免用户担心内容是否丢失。

## 需求范围

### 1. 提醒事项拖动按钮优化

提醒事项行首现有拖动按钮视觉占用过大，且破坏列表的紧凑感。

需求：

- 减弱拖动入口的视觉权重，避免每一行都显得拥挤。
- 默认状态下任务文本应成为视觉主体。
- 拖动能力仍需保留，用户需要能稳定完成同区域排序和跨时间段移动。
- 拖动入口可以在 hover、聚焦或按住任务行时出现，也可以改为更轻量的图标或区域。
- 移动端应避免依赖精细的小拖动按钮，可考虑长按进入拖动或使用更多菜单移动。

验收标准：

- 普通浏览时，任务行首不再出现明显、笨重的拖动控件。
- 鼠标用户仍能发现并使用拖动排序。
- 键盘或触屏用户不会因为按钮弱化而失去基本移动能力。

### 2. 全局保存状态

页面需要在右上角或设置入口附近提供稳定的保存状态提示。

状态包括：

- 已保存
- 保存中
- 有未保存内容

需求：

- 保存状态应常驻或在状态变化时清晰可见。
- `Ctrl+S`、失焦自动保存、延迟自动保存都应更新该状态。
- 保存气泡和动态头像可以继续作为情绪化反馈，但不应承担唯一的保存确认职责。
- 状态文案应简短，不打断用户输入。

验收标准：

- 用户修改内容后，能看到“有未保存内容”或等价状态。
- 保存触发过程中，能看到“保存中”或等价状态。
- 保存完成后，能看到“已保存”或等价状态。

### 3. 提醒事项进度展示

每个时间段标题旁显示完成进度，格式为“已完成 / 总数”。

示例：

```text
早上 2/5
中午 1/3
晚上 0/4
```

需求：

- 进度统计只统计该时间段内的提醒事项。
- 已完成数量随勾选、取消勾选、删除、移动实时更新。
- 空时间段显示 `0/0`、隐藏进度或显示空状态均可，但需要保持界面清晰。
- 总标题处可选显示全局进度，例如“提醒事项 5/12”。

验收标准：

- 用户能从每个时间段标题直接判断该段任务推进情况。
- 移动任务或清理完成项后，进度不会滞后或显示错误。

### 4. 已完成事项弱分隔

完成项移动到底部后，需要在未完成项和已完成项之间增加弱分隔。

需求：

- 仅当该时间段存在已完成事项时显示“已完成”分隔。
- 分隔视觉应弱于时间段标题，避免抢占注意力。
- 已完成事项继续保持较低视觉权重，例如文本变淡或删除线。

示例：

```text
中午 1/3

[ ] 回复设计反馈
[ ] 处理部署问题

已完成
[x] 整理会议记录
```

验收标准：

- 用户能明确区分未完成事项和已完成事项。
- 没有已完成事项时，不显示多余分隔。

### 5. 移动端分栏切换

移动端不应强行压缩桌面五栏布局，应改为分栏切换工具。

需求：

- 移动端默认打开“待办”。
- 提供顶部或底部切换入口。
- 基础分栏包括：图片、便签、待办、工作、存储。
- 快捷按钮在移动端作为独立的可展开菜单栏入口，用户展开菜单后切换到对应区域。
- 当前分栏应有明确选中态。
- 分栏切换不应导致内容丢失或未保存内容被覆盖。

建议移动端结构：

```text
图片 / 便签 / 待办 / 工作 / 存储
```

或：

```text
图片 / 便签 / 待办 / 空间
```

其中“空间”承载合并后的右侧多 tab 工作区。

移动端菜单栏要求：

- 菜单栏默认保持紧凑，不长期挤占主要内容空间。
- 展开后显示各区域入口。
- 点击菜单项后切换到对应区域，并收起菜单。
- 当前区域需要在菜单栏中有明确状态。
- 快捷按钮区作为菜单中的独立区域，不并入便签区。

验收标准：

- 手机宽度下默认能直接看到待办。
- 用户可以在主要区域之间快速切换。
- 不出现五栏被压缩到难以阅读的情况。
- 快捷按钮可以通过独立菜单入口进入，不需要先进入便签区。

### 6. 统一编辑心智模型

页面交互应统一为一套容易理解的规则。

规则：

- 文本区：点进去就写。
- 标题：双击改名。
- 条目、按钮、图片：hover 显示主要操作，右键作为高级入口。
- 所有编辑：失焦自动保存，`Ctrl+S` 立即保存。

需求：

- 文本类区域避免要求用户先进入特殊模式才能输入。
- 标题编辑入口保持一致。
- 提醒事项、快捷按钮、图片缩略图在 hover 时显示必要操作，例如更多、编辑、复制或删除。
- 右键菜单继续保留，作为高级或低频操作入口。
- 自动保存规则对所有可编辑内容保持一致的用户感知。

验收标准：

- 新用户不依赖右键也能完成主要操作。
- 熟练用户仍可使用右键完成高级操作。
- 页面内不同区域的编辑方式不互相冲突。

### 7. 今日重点区域

提醒事项区域顶部增加“今日重点”区域。

需求：

- 今日重点显示星标提醒事项。
- 今日重点位于早、中、晚分区之前。
- 今日重点只作为聚合视图，不应制造一套和原分区割裂的新任务。
- 点击或编辑今日重点中的任务时，应能影响原任务。
- 今日重点为空时，可以隐藏该区域或显示轻量空状态。
- 今日重点优先显示未完成星标事项，也可以弱化显示已完成星标事项。

示例：

```text
今日重点
★ 完成报价方案
★ 修复图片预览问题
```

验收标准：

- 用户打开待办区域后，优先看到当天最重要的事项。
- 同一条任务不会因为出现在今日重点而变成两份独立数据。
- 已完成星标事项不会抢占未完成重点事项的视觉优先级。

### 8. 星标提醒事项

提醒事项支持星标，用于标记今天特别重要的事项。

需求：

- 每条提醒事项可以添加或取消星标。
- 星标事项在本时间段内置顶，优先显示在普通未完成事项之前。
- 星标事项同步出现在“今日重点”区域。
- 星标事项完成后，在今日重点区域弱化展示。
- 取消星标后，该事项从今日重点移除，并回到普通排序规则。
- 星标不应替代完成状态，二者是独立概念。

验收标准：

- 用户能快速标记和取消重点任务。
- 星标任务在原分区和今日重点之间保持同步。
- 完成星标任务后，今日重点仍可保留回顾感，但已完成事项必须明显弱于未完成事项。

### 9. 右侧多空间工作区

将右侧 50% 的“工作空间”和“工程文件”合并为一个更大的空间区域，通过 tab 切换内容，并支持新增空间。

需求：

- 右侧区域合并为一个统一的“空间”容器。
- 顶部提供 tab 切换。
- 初始状态下只提供一个空间，名称为“工作空间”。
- 所有空间本质一致，不区分默认空间、系统空间和用户空间；差异只来自用户设置的名称和内容。
- 用户可以新增空间。
- 用户可以重命名空间。
- 用户可以删除空间；删除前必须二次弹窗确认。
- 系统必须至少保留一个空间；当只剩最后一个空间时，不允许继续删除。
- 空间删除后不可恢复。
- 每个空间独立保存内容。
- 切换空间时，当前空间内容应自动保存或保持未保存状态提示。
- 空间 tab 数量较多时，需要有可用的溢出处理，例如横向滚动、更多菜单或折叠列表。

建议初始结构：

```text
空间
[工作空间] [+]
```

空间职责：

- 工作空间：初始空间，可用于今天正在推进的任务、拆解、临时推演或长期资料。
- 新增空间：用户自定义项目、资料库、草稿、周报、接口记录等。
- 任意空间都可以通过重命名改变用途。

验收标准：

- 桌面端右侧不再同时显示两个职责相近的大文本区。
- 用户能通过 tab 在不同空间之间切换。
- 新增空间后，空间名称和内容能持久保存。
- 删除任何空间前都有二次确认，避免误删长期内容。
- 只剩最后一个空间时，删除入口不可用或明确提示至少保留一个空间。
- 空间删除后，不提供恢复入口。

## 编辑体验补充方向

空间内文本编辑体验应逐步接近苹果备忘录的自然输入感，但第一阶段保持轻量，优先增强纯文本输入。

建议能力：

- 输入 `1.` 后回车自动续编号。
- 输入 `-` 后回车自动续无序列表。
- 空编号或空列表项再次回车时退出列表。
- `Tab` 缩进当前行。
- `Shift+Tab` 取消缩进。
- 缩进后继续保持列表结构。
- 粘贴内容时尽量保留换行、缩进、编号和列表符号。

非第一阶段重点：

- 完整富文本工具栏。
- 多字体、多颜色、复杂块样式。
- 和苹果备忘录完全一致的富文本数据模型。

## 优先级建议

第一阶段：

1. 全局保存状态。
2. 提醒事项进度展示。
3. 已完成事项弱分隔。
4. 提醒事项拖动按钮视觉优化。
5. 统一 hover 操作和右键高级入口。

第二阶段：

1. 今日重点区域。
2. 星标提醒事项。
3. 右侧多空间工作区。
4. 空间内增强文本输入体验。

第三阶段：

1. 移动端分栏切换。
2. 快捷按钮在移动端的独立可展开菜单栏体验。
3. 更完整的空间管理和文本编辑能力。

## 非目标

- 不引入账号系统或云同步。
- 不把提醒事项升级为复杂项目管理系统。
- 不增加复杂标签、截止日期、负责人等字段。
- 不改变当前本地优先的使用方式。
- 不追求装饰性视觉重设计，仍保持紧凑工具风格。

## 待确认问题

无。
