# 图片列表定点粘贴与反馈设计

## 目标

为左侧图片列表的图片项右键菜单增加三个剪贴板操作：粘贴图片到当前图片上方、粘贴图片到当前图片下方、粘贴替换当前图片。所有剪贴板图片粘贴入口成功后，列表应滚动到本次粘贴对应的图片并短暂高亮。

拖放导入保持现状，不触发本设计中的滚动和高亮反馈。

## 交互设计

图片项右键菜单在现有操作中加入：

- 粘贴图片到上方
- 粘贴图片到下方
- 粘贴替换当前图片

空白区域的“粘贴图片”和全局 `Ctrl+V` 继续把图片追加到列表末尾。

粘贴成功后，目标图片卡片平滑滚动到列表可视区域中央，并播放约 700ms 的一次性高亮动画。连续粘贴或连续替换同一图片时，每次成功都重新触发动画。

权限拒绝、剪贴板没有图片、图片读取失败、IndexedDB 写入失败时不滚动、不高亮，继续使用现有动态头像气泡提示。

## 数据与组件职责

`App.vue` 继续负责剪贴板读取、图片数据生成、IndexedDB 写入、状态更新和持久化。图片粘贴请求带有以下定位信息：

- `append`：追加到末尾，用于全局粘贴和空白区域菜单。
- `before`：插入目标图片之前。
- `after`：插入目标图片之后。
- `replace`：替换目标图片内容。

`before` 和 `after` 创建新的客户端 ID 与创建时间。新图片先成功写入 IndexedDB，再插入 `state.images` 的指定位置并持久化。

`replace` 保留当前图片的 `id`、数组位置、`createdAt`、`displayWidth` 和 `displayHeight`，只替换 `src`。替换后的完整图片记录覆盖写入 IndexedDB，成功后再更新内存状态并持久化。显示尺寸不根据新图片重新计算。

`ImagePanel.vue` 负责将菜单选择转换为带目标 ID 的粘贴请求。它不直接读取剪贴板或操作图片存储。

## 滚动与高亮信号

父组件向图片列表传递独立的粘贴反馈对象，包含目标图片 ID 和每次成功都会变化的 token。使用 token 而不是只观察 ID，确保连续替换同一图片也能重复触发反馈。

列表收到新反馈后等待 Vue 完成 DOM 更新，复用现有图片卡片引用调用 `scrollIntoView`，并设置临时高亮 ID。动画结束后清除临时状态；组件卸载时清理定时器。

粘贴反馈与预览激活状态相互独立，不改变或关闭当前预览。

## 文案与视觉

新增菜单文案进入现有 i18n/state 文案结构，同时提供中文和英文。三个操作复用剪贴板图标，保持当前紧凑菜单风格。

高亮使用图片卡片已有视觉语言，通过短暂描边、阴影或背景强调实现，避免改变卡片尺寸和列表布局。动画应支持重复播放且不产生布局跳动。

## 测试范围

- 图片项菜单显示三个新增操作，并发送正确的目标 ID 和粘贴模式。
- `before`、`after` 在正确位置创建新图片。
- `replace` 只更新图片内容，保留 ID、位置、创建时间和显示尺寸。
- 全局 `Ctrl+V`、空白菜单粘贴及三个图片项粘贴操作成功后均生成反馈。
- 失败的粘贴不生成反馈。
- 图片列表收到反馈后滚动到对应卡片、添加临时高亮，并能对同一 ID 的新 token 重复触发。
- 中英文文案契约保持完整。

优先运行 `image-panel.test.ts` 和 `app-render.test.ts` 中的相关聚焦测试；由于改动涉及应用级状态、类型契约和生产打包，最终运行完整 `npm test` 与 `npm run build`。
