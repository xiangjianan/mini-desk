# 消息文案与反馈交互设计

## 背景

当前项目的消息反馈主要由 `src/state/messages.ts` 的 `MESSAGE_CATALOG`、`src/App.vue` 中的指南气泡、`src/state/defaults.ts` 中的空态文案，以及各组件中的弹窗、菜单和按钮文案共同组成。

本次设计目标是统一消息气泡的文案规则、语气强度、颜文字策略、危险确认、失败提示和删除后的撤销体验。

## 消息分类

消息按使用场景分为以下类型：

| 类型 | 场景 | 承载 |
| --- | --- | --- |
| 成功反馈 | 保存、添加图片、复制、导入导出成功 | 消息气泡 |
| 鼓励反馈 | 完成提醒事项 | 消息气泡 |
| 指南提示 | 区域使用指南、控件使用指南 | 指南气泡 |
| 空态引导 | 图片、快捷按钮、待办空态 | 页面空态或气泡 |
| 危险确认 | 删除、空间删除、导入覆盖 | 确认气泡 |
| 失败异常 | 导入失败、图片失败、剪贴板失败、链接失败 | 消息气泡 |
| 撤销提示 | 删除后、清理完成项后 | 撤销气泡 |
| 信息说明 | 关于、版本、移动端说明 | 弹窗或信息气泡 |

菜单名、按钮名、区域标签不要求扩充为多条随机文案。

## 文案规则

所有消息场景需要扩充到 10 条文案，菜单名除外。

普通消息气泡的正文控制在 15 个中文字以内。指南气泡的正文控制在 25 个中文字以内。颜文字、按钮文字、版本号、快捷键文本不计入正文长度。

图片空态文案必须体现 `Ctrl+V`，例如提示用户可以用 `Ctrl+V` 粘贴截图。

待办区指南不能写死“早上、中午、晚上”等关键词，因为这些标题可被用户编辑。指南应体现用户可以双击标题改名，也可以双击空白处新增提醒。

保存、完成、添加、复制等日常反馈可以保持活泼。删除、覆盖、失败、权限受限等消息需要更明确克制，避免弱化风险。

失败类消息按场景决定是否给下一步建议。导入、权限、图片读取、图片存储、链接打开等失败应尽量给短建议；普通复制失败可以只说明结果。

## 颜文字规则

每一个消息气泡正文后都追加一个颜文字。颜文字不计入正文长度。按钮文字不追加颜文字。

颜文字按消息分类选择：

| 分类 | 适用消息 |
| --- | --- |
| `happy` | 保存成功、添加成功、复制成功、导入导出成功 |
| `encouraging` | 完成提醒、待办激励、指南引导 |
| `warning` | 删除确认、覆盖确认、失败异常、权限受限 |
| `calm` | 撤销提示、关于、版本、移动端说明、普通信息提示 |
| `surprised` | Data URL 降级复制、明显状态变化提示 |

需要新增 `calm` 颜文字分类，用于撤销和中性说明。

## 展示时长

| 气泡类型 | 时长 |
| --- | --- |
| 普通消息气泡 | 3 秒 |
| 指南气泡 | 5 秒 |
| 撤销气泡 | 5 秒 |

现有普通气泡默认 3 秒保持不变。指南气泡从 4 秒延长到 5 秒。撤销气泡使用独立 5 秒时长。

## 危险确认

危险确认类也扩充到 10 条文案，但每条必须明确风险和后果，并使用 `warning` 颜文字。

| 场景 | 规则 | 确认按钮 | 取消按钮 |
| --- | --- | --- | --- |
| 删除图片 | 删除前确认，删除后可撤销 | 删除 | 取消 |
| 删除快捷按钮 | 删除前确认，删除后可撤销 | 删除 | 取消 |
| 删除提醒事项 | 删除前确认，删除后可撤销 | 删除 | 取消 |
| 删除空间 | 强确认，暂不提供撤销 | 删除空间 | 取消 |
| 导入覆盖 | 强确认，导入后不撤销 | 覆盖导入 | 取消 |
| 清理已完成事项 | 不确认，直接清理后可撤销 | 无 | 无 |

确认气泡不再使用通用“是/否”，改为按场景传入语义化按钮文案。

## 撤销交互

删除后显示撤销气泡。撤销气泡只有一个按钮：`撤销`。气泡持续 5 秒。

| 场景 | 撤销气泡文案 | 颜文字 |
| --- | --- | --- |
| 删除提醒事项 | 提醒已删除 | `calm` |
| 清理已完成事项 | 已清理完成项 | `calm` |
| 删除图片 | 图片已删除 | `calm` |
| 删除快捷按钮 | 快捷按钮已删除 | `calm` |

删除图片、快捷按钮、提醒事项时，用户确认后执行删除，再显示撤销气泡。清理已完成事项不做删除前确认，直接清理并显示撤销气泡。

第一版不为删除空间提供撤销，因为空间包含长期文本内容，仍保留强确认。

## 新增失败场景

第一版需要补充以下失败消息 key，全部通过消息气泡展示：

| 场景 | 文案方向 |
| --- | --- |
| 导入 JSON 解析失败 | 文件格式不正确、请检查文件 |
| 导入数据结构不合法 | 数据内容不适用、请换备份文件 |
| 图片存储失败 | 图片保存失败、请重试一次 |
| 图片读取失败 | 图片读取失败、请重新粘贴 |
| 剪贴板权限失败 | 剪贴板权限受限、请检查权限 |
| 图片复制失败 | 图片复制失败、请重试一次 |
| 文本复制失败 | 复制失败 |
| 链接打开异常 | 链接打开失败、请检查链接 |

失败类文案也扩充到 10 条，但每条语义必须稳定，不使用过度俏皮表达。

## 主要文件范围

预计实现会涉及：

- `src/state/messages.ts`：扩展消息 key、文案、颜文字分类、是否追加颜文字的统一规则。
- `src/App.vue`：指南时长、失败捕获、删除撤销、清理撤销、确认按钮文案。
- `src/components/CompanionBubble.vue`：支持语义化确认按钮和单按钮撤销气泡。
- `src/state/defaults.ts`：图片空态和相关引导文案。
- `src/__tests__/messages.test.ts` 及相关组件测试：覆盖文案数量、长度、分类、撤销和确认按钮行为。

## 验收标准

每个消息场景都有 10 条文案，菜单名除外。

普通气泡正文不超过 15 个中文字，指南气泡正文不超过 25 个中文字。颜文字不计入长度。

所有消息气泡正文后都追加与分类匹配的颜文字。

指南气泡显示 5 秒，普通气泡显示 3 秒，撤销气泡显示 5 秒。

删除图片、删除快捷按钮、删除提醒事项需要确认，确认按钮语义化。确认删除后显示可撤销气泡。

清理已完成事项不需要确认，清理后显示可撤销气泡。

删除空间继续强确认，暂不提供撤销。

失败类场景均通过消息气泡提示，并按场景提供短建议。
