# To Do List 看板

这是一个基于 Vue 3、Naive UI、TypeScript 和 Vite 构建的 To Do List 看板。页面铺满整个浏览器视口，文本和配置保存在浏览器 `localStorage` 中，图片原文保存在 IndexedDB 中。

## 使用方式

安装依赖后启动本地开发服务：

```bash
npm install
npm run dev
```

构建生产产物：

```bash
npm run build
```

运行测试：

```bash
npm test
```

## 功能说明

- 图床区支持粘贴图片、预览、缩放、拖动、删除和复制图片。
- 各区域在没有数据时会显示用途提示，方便快速理解该区域可放什么内容。
- 便签和快捷复制区域各占所在栏 50% 高度。
- 便签区、工作空间、工程文件区均使用原生 textarea，支持换行、Tab 缩进、Shift+Tab 取消缩进，并为缩进行显示小圆点提示。
- 快捷按钮支持复制 URL 或复制文本，按钮只显示标题。
- 快捷按钮通过图标新增按钮打开模态框创建，不在页面中常驻新增表单。
- 新增/编辑快捷按钮时，通过勾选框设置是否为链接属性。
- 快捷按钮采用带间距的白色线框按钮。
- 快捷按钮支持右键编辑、隐藏、取消隐藏和删除。
- 待办事项区分为早、中、晚三块，支持点击空白处新增、单击编辑、回车续写、完成、右键菜单删除、拖拽排序和跨区域拖动。
- 每个待办模块右上角支持一键清除已完成事项，按钮使用斜向刷子图标。
- 空白新增的待办如果没有输入文字，离开编辑状态时会自动丢弃，不会保存。
- 标记完成的待办事项会自动移动到所在区域底部。
- 所有删除操作都会先弹出二次确认。
- 支持白天模式和黑夜模式切换。
- 支持导入、导出 JSON，覆盖页面的主题、标题、图片、便签、快捷按钮、提醒事项、工作空间和工程文件数据。
- 图片原文保存在浏览器 IndexedDB 中，页面状态在 `localStorage` 中只保存图片元数据，避免多张大图触发本地存储上限。
- 页面为紧凑线框布局，主要区域之间只使用单条分隔线。

## 图床快捷键

点击图片进入预览后：

- `Delete`：删除当前图片。
- `Enter`：复制当前图片。
- `Space`：关闭预览。
- 鼠标滚轮或触控板：放大、缩小图片。
- 鼠标按住拖动：移动图片。

## 保存逻辑

- `Ctrl+S`：立即保存全部数据，只显示视频头像和保存气泡。
- 文本输入停止 3 秒后自动保存。
- 便签区、工作空间、工程文件区输入后 3 秒延时保存；如果 3 秒内失焦，则立即保存并取消这次延时保存。
- 文本区域失焦 1 秒后自动保存。
- 如果失焦后 1 秒内聚焦到另一个文本区域，则取消这次失焦保存。
- 文本区域聚焦时，右下角显示 `static/video/hermes.gif` 动态头像。
- 提醒事项输入时，动态头像显示在提醒事项区域内。
- 没有 textarea 聚焦时，`Ctrl+S` 保存会在页面右下角显示动态头像和保存气泡。
- 动态头像隐藏时不会暂停播放。
- textarea 失焦后，动态头像位置恢复到屏幕右下角。
- 动态头像和保存气泡出现、消失时带渐变效果。
- 保存时显示“已保存”语义的随机 NPC 对话气泡消息，3 秒后消失。

## 数据位置

数据保存在浏览器 `localStorage` 中，键名为：

```text
todo-board-state-v1
```

图片原文保存在同站点 IndexedDB 的 `todo-board-images-v1` 数据库中。清空浏览器站点数据会清除看板内容和图片；只手动删除 `localStorage` 键会清除看板状态，但可能保留 IndexedDB 中的图片原文。

## 项目结构

- `src/App.vue`：看板主入口和跨组件事件编排。
- `src/components/`：图床、快捷按钮、待办、文本区、预览层、设置入口和保存动效组件。
- `src/state/`：默认状态、localStorage/IndexedDB 存取、导入导出归一化和待办领域逻辑。
- `src/utils/`：textarea 缩进和文本行工具。
- `src/__tests__/`：状态兼容、待办逻辑和 Vue 渲染契约测试。
