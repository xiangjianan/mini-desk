# To Do List 看板

当前版本：`1.0.17`

给老婆做的 todolist 看板。它基于 Vue 3、Naive UI、TypeScript 和 Vite 构建，是一个本地优先的桌面端工作看板，用来集中整理截图、便签、提醒事项、快捷链接和多个工作空间。页面状态保存在浏览器 `localStorage`，图片原文保存在 IndexedDB。

## 在线地址

- GitHub Pages: <https://xiangjianan.github.io/todolist/>
- Cloudflare Pages: <https://todo.helloxjn.com/>

## 快速开始

安装依赖：

```bash
npm install
```

启动本地开发服务：

```bash
npm run dev
```

如果需要在局域网内用其他设备访问：

```bash
npm run dev -- --host 0.0.0.0
```

构建生产产物：

```bash
npm run build
```

运行测试：

```bash
npm test
```

## 功能概览

### 图床

- 支持 `Ctrl+V` 粘贴截图。
- 支持把浏览器外部图片拖入图床。
- 图片原文写入 IndexedDB，页面状态只保存图片元数据。
- 图片列表支持缩略图预览、拖拽排序、右键菜单、删除和复制。
- 预览模式支持点击空白处关闭、侧边栏切换图片、右键菜单、滚轮缩放和拖动平移。
- 预览快捷键：`Enter` 复制、`Delete` 或 `Backspace` 删除、`Space` 或 `Esc` 关闭、方向键切换图片。

### 便签

- 使用原生 textarea，点进去即可输入。
- 支持换行、Tab 缩进、Shift+Tab 取消缩进。
- 删除行文字后继续按删除键，可以取消该行缩进。
- 文本区为空时，聚焦会显示 Tips 气泡和动态头像；非空时只显示动态头像。

### 快捷链接

- 支持链接型按钮和复制文本型按钮。
- 复制文本型按钮支持多行内容。
- 按钮点击后打开链接或复制文本。
- 右键菜单支持编辑、隐藏、显示和删除。
- 顶部三点菜单支持新增按钮、显示隐藏项、收起隐藏项和 Tips。
- 支持拖拽排序。

### 提醒事项

- 按早上、中午、晚上三个区域组织提醒事项。
- 单击空白处新增提醒事项，单击条目即可编辑。
- 支持回车续写、完成、删除、复制文本、拖拽排序和跨区域移动。
- 每个时间段标题旁显示完成进度，例如 `2/5`。
- 已完成提醒事项默认隐藏，可按区域单独显示或隐藏。
- 完成项前会显示弱分隔标题“已完成”。
- 支持清理已完成事项，清理前会弹出二次确认气泡。

### 今日重点

- 星标提醒事项会在所属区域置顶，并同步出现在“今日重点”区域。
- 今日重点会显示未完成和已完成的星标事项，已完成项弱化展示。
- 今日重点标题支持双击改名，默认标题为 `❗️ 今日重点`。
- 今日重点中的条目不参与拖拽排序，排序仍以原提醒事项区域为准。

### 工作空间

- 右侧区域合并为工作空间 Tab。
- 初始只有一个“工作空间”，系统至少保留一个空间。
- 支持新增空间、双击重命名、右键删除、拖拽排序。
- 删除空间前会二次确认；删除后不可恢复。
- 空间过多时，Tab 栏支持横向滚动。
- 工作空间文本区点进去即可输入，支持与便签一致的缩进体验。

### 设置与反馈

- 顶部显示保存状态：已保存、保存中、有未保存内容。
- 设置菜单支持数据导入、数据导出、提建议、关于和版本状态。
- “提建议”会打开 GitHub issue 创建页面。
- “关于”通过消息气泡显示项目简介，并提供可点击的 GitHub 项目链接。
- 支持浅色和深色主题切换，主题选择会自动保存。

## 消息气泡与 Tips

- 保存、删除确认、导入导出、复制反馈、区域 Tips 都通过统一的消息气泡展示。
- 气泡出现和消失有过渡动画。
- 鼠标悬浮在消息气泡上时，会暂停自动消失计时；鼠标移开后继续计时。
- 图床、快捷链接和提醒事项空白区域的 Tips 在显示期间不会因重复点击而快速切换。
- 切换到其他区域会取消当前 Tips，新区域可以显示自己的 Tips。
- 动态头像最多显示 10 秒，超过后会渐隐。

## 保存逻辑

- `Ctrl+S` 会立即保存并显示保存气泡。
- 文本输入停止 3 秒后自动保存。
- 文本区域失焦后会自动保存。
- 导入、删除、清理已完成等高风险操作都会先确认。
- 删除后的撤销入口已取消，删除确认后即生效。

## 数据位置

看板状态保存在浏览器 `localStorage`：

```text
todo-board-state-v1
```

图片原文保存在同站点 IndexedDB：

```text
todo-board-images-v1
```

清空浏览器站点数据会清除看板内容和图片。只删除 `localStorage` 键会清除看板状态，但 IndexedDB 中可能仍保留图片原文。

## 移动端说明

当前看板主要为桌面端工作流设计。移动端会显示引导页，提示用户在电脑浏览器打开，以获得完整体验。

## 部署

GitHub Pages 通过 GitHub Actions 自动部署。构建时会设置仓库子路径：

```bash
VITE_BASE=/todolist/ npm run build
```

Cloudflare Pages 使用 `dist` 目录部署，项目名为 `todolist`。

Vite 的 `base` 由环境变量控制：

```ts
base: process.env.VITE_BASE ?? "/"
```

## 项目结构

- `src/App.vue`：看板主入口，负责跨组件状态编排、保存、导入导出、消息气泡和快捷键。
- `src/components/`：图床、图片预览、快捷链接、提醒事项、文本区、工作空间、设置菜单和消息气泡组件。
- `src/state/`：默认状态、消息文案、版本信息、localStorage/IndexedDB 存取、导入导出归一化和待办领域逻辑。
- `src/utils/`：文本编辑缩进与行处理工具。
- `src/__tests__/`：组件渲染、状态兼容、交互契约、部署配置和消息文案测试。
- `.github/workflows/`：GitHub Pages 自动部署配置。
